# 安全政策

本项目是本地优先的VK BaoXiao Agent，没有托管后端。邮箱模式会连接用户配置的 IMAP 服务并下载邮件中的发票链接；图片是否交给联网视觉模型由宿主 Agent 决定，必须在运行前说明并取得授权。

## 支持版本

仅最新 `main` 分支接受安全更新。

## 报告漏洞

请**不要**在公开 Issue 中披露安全漏洞。

- 通过 GitHub 仓库的 **Security → Report a vulnerability** 私密提交；
- 或邮件联系维护者（见 `README.md` 的「作者与致谢」）。

我们会在 7 个工作日内响应。

## 凭据与敏感数据

- 邮箱授权码、账户配置保存在 `.env` 与 `config/package-config.js`，已被 `.gitignore` 忽略，**严禁提交入库**。
- 发票原件、扫描缓存、归档结果位于 `scan-results/`、`archive/`，同样被忽略。
- 提交前请运行 `npm run check` 并确认 `git status` 中无上述文件。

## 已知边界

- AI 识别 / PDF 解析 / 邮件链接下载都可能出错，正式报销、入账、纳税申报、审计材料**必须人工复核**。
- 本项目不提供财务、税务、法律、审计等专业意见。

## 默认安全边界

- IMAP 默认校验证书。
- 邮件链接默认只允许公网 HTTPS，并限制重定向和响应大小。
- 清理操作只能作用于声明的工作根目录下的子目录。
- 规则候选只有取得 rules.write 授权后才能进入正式分类规则。
